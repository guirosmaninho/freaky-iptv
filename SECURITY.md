# Security policy

## Supported versions

Security fixes are applied to the latest release on the `main` branch.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability or include credentials,
playlist URLs, EPG URLs, tokens, logs, or personal viewing data in a report.

Use [GitHub private vulnerability reporting](https://github.com/guirosmaninho/freaky-iptv/security/advisories/new).
Include the affected version, reproduction conditions, impact, and a minimal
proof of concept with secrets removed.

For ordinary application bugs, use the
[public issue tracker](https://github.com/guirosmaninho/freaky-iptv/issues/new).
